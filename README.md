## Возможности

### 📊 Дашборд
- Баланс активов и пассивов с визуальной шкалой
- Доходы и расходы за выбранный месяц
- Сводка кешбека по банкам с детализацией по правилам
- Количество активных долгов
- Последние операции

### 💳 Доходы, Расходы и Переводы
- Три типа операций: доход, расход, перевод между счетами
- Поддержка кредитных/ипотечных платежей с разделением на тело кредита и проценты
- Выбор категории с иерархией (родитель → подкатегория)
- Автоподстановка MCC-кода из категории
- Флаг "Не рассчитывать кешбек" для отдельных расходов
- Повтор операции одним нажатием

### 🏦 Счета
- Гибкая система счетов с привязкой к банку и типу счёта
- Типы счетов: дебетовые, кредитные, ипотека, накопительные, наличные
- Автоматический пересчёт баланса при добавлении/редактировании/удалении операций
- Группировка по типам на странице баланса
- Фильтрация истории по конкретному счёту

### 🏛️ Банки
- Справочник банков с иконками, цветами и порядком сортировки
- Встроенные пресеты: Сбербанк, Т-Банк, Альфа-Банк, ВТБ, Газпромбанк, Почта Банк, Озон Банк, Яндекс Банк, МТС Банк, Финуслуги
- Drag-and-drop сортировка

### 📂 Категории
- Двухуровневая иерархия: родительские категории → подкатегории
- 13 предустановленных категорий расходов с подкатегориями (супермаркеты, коммуналка, общепит, транспорт и т.д.)
- 6 категорий доходов
- MCC-коды на категориях для автоподстановки
- Drag-and-drop сортировка

### 💰 Кешбек-калькулятор
- Расчёт кешбека по правилам банков на основе транзакций
- Два способа привязки правил: по категории или по MCC-коду
- Детальный расчёт на дашборде с группировкой по банкам
- Отображение кешбека напротив каждой транзакции в истории
- Поддержка временных акций (дата начала/окончания)
- 9 банков с предустановленными пресетами правил кешбека

### 🎯 Бюджеты
- Месячные лимиты расходов по категориям
- Привязка к месяцу и году

### 💸 Долги
- Два типа: "занял" (lent) и "взял" (borrowed)
- Статусы: активный, закрытый, просроченный
- Процентная ставка и срок возврата
- График платежей по долгу

### 👨‍👩‍👧‍👦 Семейный учёт
- Несколько участников с именами, аватарами и цветами
- Привязка счетов и транзакций к члену семьи

### 📋 История операций
- Группировка по дням
- Фильтры: по счёту, по типу (все/доходы/расходы/переводы)
- Фильтры по дате: месяц, прошлый месяц, 90 дней, год, прошлый год, произвольный период
- Редактирование и удаление транзакций
- Повтор операции

### 📈 Отчёты
- Гистограмма доходов и расходов за период (1, 3, 6, 12 месяцев)
- Круговая диаграмма структуры расходов за текущий месяц
- Таблица остатков по всем счетам

### ⚙️ Настройки
- Управление категориями (с drag-and-drop)
- Управление типами счетов
- Управление банками
- Управление счетами
- Управление участниками семьи
- Управление бюджетами
- Управление правилами кешбека для каждого банка
- **Экспорт и импорт всех данных** (JSON-дампа всех 11 таблиц)

### 📱 PWA
- Установка на домашний экран
- Работа в офлайне
- Автообновление service worker
- Адаптивный мобильный дизайн (mobile-first, max-w-lg)

## Структура базы данных

Приложение использует **IndexedDB** через **Dexie 4**. Всего **11 таблиц**, текущая версия схемы — **19**.

| Таблица | Назначение | Ключевые индексы |
|---|---|---|
| `transactions` | Операции (доходы, расходы, переводы) | `date`, `categoryId`, `type`, `accountId`, `transferToAccountId`, `familyMemberId` |
| `categories` | Категории и подкатегории | `type`, `order`, `parentId` |
| `budgets` | Месячные лимиты по категориям | `[month+year]`, `categoryId` |
| `familyMembers` | Участники семейного учёта | — |
| `accountTypes` | Типы счетов (дебетовый, кредитный, ипотека) | `order` |
| `banks` | Банки | `name`, `order` |
| `accounts` | Счета (привязаны к банку, типу и участнику) | `familyMemberId`, `order`, `bankId`, `typeId` |
| `debts` | Долги | `status`, `familyMemberId` |
| `debtPayments` | Платежи по долгам | `debtId` |
| `cashbacks` | Правила кешбека банков | `bankId`, `&[bankId+name]` (unique) |
| `accountCashbacks` | Привязка правил кешбека к счетам | `accountId`, `cashbackId`, `categoryId`, `[accountId+cashbackId]` (compound) |

### Ключевые типы и интерфейсы (src/db/db.ts)

```typescript
interface Transaction {
  id?: number
  amount: number
  description: string
  categoryId?: number
  accountId: number
  familyMemberId: number
  date: Date
  type: 'income' | 'expense' | 'transfer'
  principalAmount?: number | null   // тело кредита (для ипотечных платежей)
  interestAmount?: number | null    // проценты (для ипотечных платежей)
  transferToAccountId?: number      // счёт получателя (для переводов)
  mcc?: number                      // MCC-код транзакции
  noCashback?: boolean              // флаг отключения кешбека
  createdAt: Date
}

interface Account {
  id?: number
  name: string
  typeId: number      // ссылка на AccountType
  bankId: number      // ссылка на Bank
  currency: string
  balance: number
  icon: string
  color: string
  familyMemberId: number
  order: number
  createdAt: Date
}

interface AccountType {
  id?: number
  name: string
  icon: string
  color: string
  order: number
  kind: 'regular' | 'credit' | 'mortgage'
}

interface Category {
  id?: number
  name: string
  icon: string
  color: string
  type: 'income' | 'expense'
  order: number
  parentId?: number    // null = корневая категория
  mcc?: number
}
```

### Версионирование схемы

База данных имеет 19 версий схемы (миграции в `src/db/db.ts`). При обновлении приложения Dexie автоматически применяет миграции — данные пользователя сохраняются.

### Инициализация (seed)

При первом запуске (когда таблицы пусты) автоматически создаются:
- 7 типов счетов (дебетовая карта, срочный счёт, накопительный, кредитка, ипотека, кредит, наличные)
- 10 банков
- 13 категорий расходов + подкатегории
- 6 категорий доходов
- Default-участник "Я"
- Пресеты кешбека для 9 банков

## Деплой

### GitHub Pages

Деплой происходит автоматически через GitHub Actions при пуше в ветку `main`:

1. **Build**: установка зависимостей → сборка (`npm run build`) → артефакт `dist/`
2. **Deploy**: публикация на GitHub Pages через `actions/deploy-pages`

Конфигурация PWA учитывает подпуть `/MoneyCheck/`:
- `base: '/MoneyCheck/'` в `vite.config.ts`
- `scope` и `start_url` манифеста: `/MoneyCheck/`
- `navigateFallback` Workbox: `/MoneyCheck/`

### Важные замечания

- **Все данные в браузере**: при смене устройства или очистке кеша данные теряются. Регулярно используйте **экспорт данных** (Настройки → Управление данными → Экспорт).
- **Service Worker**: автообновление (`registerType: 'autoUpdate'`) — при новой версии приложение обновляется без участия пользователя.

## Архитектура

### Роутинг

Используется **HashRouter** с 6 маршрутами:

| Путь | Страница | Назначение |
|---|---|---|
| `/` | Dashboard | Главная с балансом, отчётом за месяц |
| `/add` | AddExpense | Добавление/редактирование операции |
| `/balance` | Balance | Просмотр баланса по всем счетам |
| `/transactions` | Transactions | История операций с фильтрами |
| `/reports` | Reports | Графики и аналитика |
| `/settings` | Settings | Настройки (все менеджеры) |

Страница Settings использует SPA-style переключение вьюх (`useState<View>`), а не подмаршруты.

### Реактивность

Два паттерна сосуществуют:

1. **`useLiveQuery`** (Dexie React hooks) — для чтения данных. Компоненты автоматически перерендериваются при изменении IndexedDB. Используется в хуках `src/hooks/useDb.ts`.
2. **Асинхронные CRUD-функции** — для записи. Прямые вызовы Dexie в `src/db/*.ts`. После записи Dexie уведомляет `useLiveQuery` об изменениях.

### AppShell (Layout)

Mobile-first дизайн:
- Контейнер `max-w-lg mx-auto` — центрирование на десктопе
- `h-dvh` — полная высота viewport
- Нижняя навигация с 6 пунктами (фиксированная, `bottom: 0`)
- Safe area insets для современных устройств
- Скролл внутри контейнера

### Локализация

- Весь интерфейс на **русском языке**
- Форматирование валюты: **RUB, ru-RU** (с пробелами-разделителями)
- Форматирование дат: **D MMM YYYY** (например, "15 мар 2025")
- Месяцы: сокращённые русские названия (Янв, Фев, Мар...)
