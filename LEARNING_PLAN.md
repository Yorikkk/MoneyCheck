### Шаг 1 — JSX, props, компоненты
**Когда:** 15 июня 2026
**Что делали:**
- Добавили проп `className` в AppShell с дефолтным значением `''`
- Склеили его с базовыми классами через шаблонную строку
- Передали `className="relative"` из App.tsx
**Что узнали:**
- JSX — это JS-разметка: `className` вместо `class`, `{}` для выражений
- Компонент — функция, возвращающая JSX
- Props — аргумент функции, деструктуризация на входе
- `children` — всё, что внутри тега компонента
- Шаблонные строки `` `${var}` `` позволяют вставлять переменные в строку

### Шаг 2 — useState, события
**Когда:** 15 июня 2026
**Что делали:**
- Добавили состояние `place` через `useState('')`
- Добавили input "Место покупки" с `onChange`
- Добавили валидацию: `place.length > 30 → ошибка`
- Добавили счётчик `place.length/30`
- Заблокировали кнопку при ошибке через `!!placeError`
**Что узнали:**
- `useState` возвращает [значение, setter]
- setter вызывает перерендер компонента
- Вычисляемые значения (константы) не требуют useState — достаточно const
- `!!str` превращает строку в boolean
- Условный рендеринг: `{condition && <JSX>}`
